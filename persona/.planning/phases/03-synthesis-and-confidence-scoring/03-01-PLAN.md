The Synthesis Protocol section must contain these subsections in order:

### 1. Input Collection
- Instruct Claude to use Glob to find all `persona-reviews/*.json` files
- Read each JSON file using the Read tool
- If `persona-reviews/` does not exist or is empty, output: "No persona reviews found. Run `/persona:review` first."

- Flatten all findings from all persona JSON files into a single working list

### 2. Deduplication (per D-04, D-05, SYNT-02, SYNT-03)
Instructions for Claude to group findings semantically:
```
Group findings that describe the SAME issue in the SAME file. Two findings are "the same issue" if:
- They reference the same file (exact path match)
- They describe semantically similar problems (e.g., "missing error handling" and "no try-catch around async call" about the same function)

Do NOT group findings that happen to be in the same file but describe different issues.
When in doubt, keep findings separate rather than merging them.
```

For each deduplicated group:
- Use the most detailed description as the primary issue text
- Combine unique recommendations from all contributing personas
- List all contributing personas with their individual confidence scores: "Flagged by: ThePrimeagen (85), DHH (75)"
- Preserve each persona's original reasoning verbatim -- do NOT merge reasoning into a bland average (per Pitfall 3 in research)

### 3. Confidence Boosting (per D-06)
Formula: `boosted_confidence = min(99, max_individual_confidence + (10 * (persona_count - 1)))`
- Single persona: original confidence (no boost)
- Two personas agree: +10
- Three personas agree: +20
- Capped at 99 (never auto-100)
- Show the boosted score as the finding's confidence, with individual scores in the attribution

### 4. Disagreement Detection (per D-07, D-08, SYNT-05)
Detection rules:
1. **Severity conflict:** Two personas flag the same file + issue area but assign different severities (one says critical, another says suggestion)
2. **Approach conflict:** One persona recommends X for a file, another persona explicitly cautions against X for the same file

Disagreements appear in a dedicated `### Disagreements` section. Each disagreement shows:
- The file and issue area
- Each persona's position with their severity, confidence, and reasoning quote

### 5. Confidence Threshold Filtering (per D-12, CONF-02)
- Parse `--min-confidence N` from skill arguments (default: 30)
- NEVER filter critical-severity findings regardless of confidence (per Pitfall 4 in research)
- Filter warning and suggestion findings below the threshold
- At the bottom of output, show: "N findings hidden below confidence threshold of {threshold}. Use `--min-confidence 0` to see all."

### 6. Output Format (per D-09, D-10, D-11, SYNT-04)
Provide the EXACT output template that Claude must follow:

```markdown
## Persona Review Synthesis

**{N} personas reviewed `{target}`**
**Summary: {N} critical, {N} warnings, {N} suggestions** (after deduplication)
**Confidence threshold: {threshold}** ({N} findings hidden)

---

### Critical ({count})

#### 1. {issue title}
- **Confidence:** {score} {if boosted: "(boosted -- flagged by N personas)"}
- **File:** {file}:{line}
- **Issue:** {description}
- **Recommendation:** {combined recommendations}
- **Flagged by:** {Persona1} ({score1}), {Persona2} ({score2})
- **Reasoning:**
  - *{Persona1}:* "{reasoning}"
  - *{Persona2}:* "{reasoning}"

### Warnings ({count})
[same format]

### Suggestions ({count})
[same format]

---

### Disagreements ({count})

#### 1. [{file}] {issue area}
- **{Persona1}** ({severity}, confidence {score}): "{reasoning}"
- **{Persona2}** ({severity}, confidence {score}): "{reasoning}"

---

*{N} findings hidden below confidence threshold of {threshold}. Use `--min-confidence 0` to see all.*
```

```markdown
# Synthesize Persona Reviews

Read all persona review JSON files from `persona-reviews/` and produce a unified, deduplicated review following the Synthesis Protocol.

## Arguments

Parse `$ARGUMENTS` for:
- `--min-confidence N`: Minimum confidence threshold for filtering findings (default: 30). Critical-severity findings are never filtered.

## Steps

1. Check that `persona-reviews/` directory exists at the project root
2. Use Glob to find all `persona-reviews/*.json` files
3. If no JSON files found, tell the user: "No persona reviews found. Run `/persona:review` first."
4. Read each JSON file using the Read tool
5. Follow the **Synthesis Protocol** in `skills/review/reference.md` to:
   - Deduplicate findings by file + semantic similarity
   - Apply confidence boosting for multi-persona agreement
   - Detect and surface cross-persona disagreements
   - Filter findings below the confidence threshold (except critical)
   - Format output as severity-grouped markdown
6. Present the synthesized review to the user
7. Write the synthesis output to `persona-reviews/SYNTHESIS.md`
```
