Brief description: Multi-persona code review orchestrator for Claude Code. Get feedback from 14 distinct expert perspectives -- each with their own philosophy, priorities, and blind spots.

**## Installation**
```
/plugin install persona@ai-stuff
```

**## Quick Start**
Show 3 usage examples:
- `/persona:review src/auth.ts` -- review a specific file
- `/persona:review --staged` or just `/persona:review` -- review staged changes
- `/persona:review src/api/ --only theprimeagen,dhh --gilfoyle` -- select specific personas with maximum intensity

**## Usage**
Document all flags:
- `[target]` -- file, directory, or glob to review (defaults to staged changes)

- `--min-confidence N` -- filter findings below confidence threshold (default: 30, critical findings never filtered)

**## Custom Personas**
Instructions for creating custom personas:
1. Copy `agents/template.md` to `agents/your-persona-name.md`
2. Fill in the frontmatter (name, description)
3. Write your persona's voice, beliefs, and focus areas
4. Keep the standard sections (Project Conventions, Bash Usage, Review Output Format, Project Stack Constraint, Gilfoyle Mode, JSON Output Mode, Memory Curation) unchanged
5. The persona will be automatically available in the next review

Note: The `name` field in frontmatter must be kebab-case and match the filename (without .md). The JSON Output Mode section's `persona` field must also match.

**## How It Works**
Brief 3-4 sentence explanation: The orchestrator dispatches all selected personas in parallel as Claude Code subagents. Each persona reads the code through its unique philosophical lens and returns structured JSON findings. The synthesis engine deduplicates findings, boosts confidence when multiple personas agree, surfaces disagreements, and produces a unified review.
