```json
{
  "hooks": {
    "SubagentStart": [
      {
        "matcher": "theprimeagen|dhh|chris-coyier|dan-abramov|evan-you|kent-c-dodds|lee-robinson|matt-mullenweg|matt-pocock|rich-harris|scott-tolinski|tanner-linsley|theo-browne|wes-bos",

          {
            "type": "command",
            "command": "echo \"[persona] Starting review: $(echo $HOOK_INPUT | jq -r .agent_type)\" >&2"
          }
        ]
      }
    ],
    "SubagentStop": [
      {
        "matcher": "theprimeagen|dhh|chris-coyier|dan-abramov|evan-you|kent-c-dodds|lee-robinson|matt-mullenweg|matt-pocock|rich-harris|scott-tolinski|tanner-linsley|theo-browne|wes-bos",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"[persona] Finished review: $(echo $HOOK_INPUT | jq -r .agent_type)\" >&2"
          }
        ]
      }
    ]
  }
}
```

IMPORTANT implementation notes:
- Hooks receive input as JSON on **stdin**, NOT via environment variable. The inline command must read from stdin. Use: `jq -r .agent_type` (reads stdin by default) — do NOT reference `$HOOK_INPUT` or any env var.
- Output progress messages to **stderr** (>&2) so they appear in the user's console. Stdout is parsed as JSON response by Claude Code — random text on stdout will cause errors.
- Use command-type hooks only (zero LLM cost, per CLAUDE.md).
- Matcher is a single pipe-separated regex string containing all 14 exact agent names. This ensures hooks only fire for persona agents, not built-in agents (Bash, Explore, Plan).
- All 14 names must be lowercase kebab-case matching the `name:` frontmatter in agents/*.md exactly (per Pitfall 12 — matchers are case-sensitive).
- Commands are inline bash — no script file references (per Pitfall 4 — Windows compatibility).
- Since stdin can only be read once, and we need agent_type from stdin, pipe stdin through jq in a single pipeline. Pattern: `jq -r '.agent_type' | xargs -I{} echo "[persona] Starting review: {}" >&2`
- Do NOT add `"async": true` — that is only needed for SessionStart hooks (per Pitfall 4). SubagentStart/Stop do not need it.
