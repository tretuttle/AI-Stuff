**Create `skills/review/SKILL.md`** with the following structure. Keep under 500 lines total.

**Frontmatter (per D-09, D-17, ORCH-05):**
```yaml
---
name: review
description: "Dispatch multi-persona code review against targeted files or changes"
argument-hint: "[file/dir/glob] [--only name1,name2] [--gilfoyle]"
---
```

Do NOT include `context: fork` (ORCH-05, D-17). Do NOT include `disable-model-invocation: true`.

**Body structure -- 6 sections:**

**1. Overview** (2-3 lines)
State that this skill dispatches persona agents for code review. Reference `reference.md` for the persona roster, JSON schema, and Gilfoyle mode block.

**2. Argument Parsing** (per D-04, D-05, ORCH-03)
Instruct Claude to parse `$ARGUMENTS` as follows:
- Split on whitespace
- Extract `--only <comma-separated-names>` if present. Names can be display names or agent names (map display names to agent names using the roster in reference.md)
- Extract `--gilfoyle` flag if present
- Everything remaining is the review target (file path, directory, or glob pattern)
- If no review target provided, use staged changes: run `git diff --staged` and use that output as the review content
- Examples: `$ARGUMENTS` = `src/auth.ts` -> target=src/auth.ts, all personas, no gilfoyle. `$ARGUMENTS` = `packages/convex/ --only "Matt Pocock,Theo Browne"` -> target=packages/convex/, only matt-pocock and theo-browne. `$ARGUMENTS` = `` (empty) -> target=staged diff, all personas.

**3. Pre-Dispatch Confirmation** (per D-10)
Before dispatching, print a confirmation showing:
- "Personas:" followed by the list of persona display names that will run
- "Target:" followed by the review target (file path or "staged changes")
- "Gilfoyle mode:" yes/no
- Then proceed with dispatch (no user approval gate needed -- just informational display)

**4. Setup** (per D-08)
Before dispatching personas, create the output directory:
- Run `mkdir -p persona-reviews` at the project root
- This ensures the directory exists for writing output files

**5. Dispatch** (per D-02, D-18, ORCH-02)
This is the critical section. Include these EXACT instructions:

"You MUST use the Task tool to dispatch each persona subagent. Do NOT review the code yourself. Do NOT skip delegation."

For each selected persona (all 14 if no --only filter, or the filtered subset per D-01):
- Use the Task tool with agent name set to the persona's kebab-case agent name from the roster in reference.md
- The task prompt MUST include:
  a. The review target (file path, directory, glob, or the staged diff content)
  b. Instruction to return findings as a JSON object matching the schema in reference.md
  c. Instruction to set `gilfoyleMode` field to true/false based on the flag
  d. If --gilfoyle is active, append the Gilfoyle mode block from reference.md
- Dispatch ALL selected personas in parallel -- issue all Task tool calls before waiting for any results
- Do NOT dispatch sequentially

**6. Collection and Summary** (per D-06, D-07, D-11, D-12, ORCH-04)
After each persona task completes:
- Parse the persona's returned content as JSON
- Write it to `persona-reviews/{agent-name}.json` using the Write tool (per D-06, D-08)
- Print a completion message: "{Display Name} complete" (per D-11)

After ALL personas complete:
- Read all JSON files from `persona-reviews/`
- Count findings by severity across all personas
- Print a ranked severity summary (per D-12):
  - Total criticals, warnings, suggestions
  - List critical findings first with persona attribution
  - Then warnings, then suggestions
- Note: Full synthesis/deduplication is Phase 3. This summary is a simple count + listing.

**Create `.gitignore`** at the project root (or append if it exists):
Add a single line: `persona-reviews/`
This excludes ephemeral review output from version control.
